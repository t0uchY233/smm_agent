# SMM Conductor — авто-роутер для контент-сессий

Срабатывает один раз на первом сообщении пользователя. Молча определяет, что хочет пользователь, и вызывает нужный скилл. **Не задаёт уточняющих вопросов на этом этапе** — все уточнения делает уже целевой скилл (`/video-script` или `/publish-from-script`).

## Классификатор

Применяй сверху вниз — первое подошедшее правило выигрывает.

### 1. YouTube URL

**Признак:** в сообщении есть подстрока `youtube.com/watch?v=` или `youtu.be/`.

**Действие:** следовать `.codex/skills/publish-from-script/SKILL.md`, передав весь URL как аргумент. Все остальные слова из сообщения игнорировать (это и так комментарий пользователя). Дальнейший flow определяет сам `/publish-from-script`.

### 2. Тема для видео

**Признаки темы (достаточно одного):**
- Существительное-концепт из домена пользователя (бизнес, управление, закупки, ТОС, нефтегаз, бурение, КРС, тендеры, ИИ-агенты, маркетинг, продажи, финансы)
- Вопрос «как / почему / что / сколько» обращённый к аудитории, а НЕ к Codex («как уволить зама», «почему 90% закупок проигрывают», «что делать когда поставщик кинул»)
- Провокационный заголовок («3 ошибки руководителя», «уволь финдира», «закупщик не нужен»)
- Описание кейса с цифрами и контекстом («металлургический завод сэкономил 12 млн за полгода», «нефтянка 1200 скважин КРС 2.8 млрд бюджет»)
- Явный префикс «Тема:», «тема —», «давай сценарий про», «сценарий о…», «сделай ролик про», «нужен сценарий…»

**Действие:** следовать `.codex/skills/video-script/SKILL.md`, передав всё сообщение целиком как `args` (целевой скилл сам разберёт топик и доп. контекст). Если в сообщении есть дата/время публикации (например `2026-05-01 14:00`) — это express mode для `/video-script`, ничего дополнительно делать не надо, скилл подхватит.

### 3. Прочее — без роутинга

**Признаки (любой):**
- Сообщение начинается с `/` (это явная команда — выполнить как есть)
- Содержит слова про артефакты репозитория: «файл», «тул», «скрипт», «n8n», «webhook», «sheets», «git», «коммит», «merge», «push», «PR», или конкретное имя файла/пути (`*.json`, `*.md`, `tools/`, `.codex/`)
- Вопросы о состоянии работы: «что сделано», «какие изменения», «покажи diff», «как работает X»
- Просьбы исправить код, отрефакторить, добавить feature к коду
- Приветствие, благодарность, «ок», «понятно», «продолжай»

**Действие:** не звать `/video-script` и не звать `/publish-from-script`. Просто отвечать на сообщение нормально — это техническое взаимодействие с проектом, а не запрос на контент.

## Жёсткие правила

1. **Срабатываю только на первом сообщении сессии.** На последующих сообщениях не вмешиваюсь — дальше пользователь и целевой скилл общаются напрямую.
2. **Не переспрашиваю «это тема для видео?».** Если классификатор сработал → сразу инвок целевого скилла. Если не сработал → обычная обработка. Уточнения делает целевой скилл.
3. **Не печатаю промежуточный комментарий типа «Я понял что это тема, запускаю /video-script…».** Просто выполняю нужный workflow. Целевой скилл сам себя представит.
4. **Сохраняю весь текст пользователя.** Передаю целиком, не пересказываю и не сокращаю — там может быть кейс/цифры/детали для сценария.

## Анти-примеры (что НЕ есть тема)

| Сообщение | Классификация | Почему |
|---|---|---|
| «Привет» | прочее | приветствие, не контент |
| «как там с git, что осталось докоммитить» | прочее | про репо, не про аудиторию |
| «открой n8n.json» | прочее | команда на файл |
| «Поправь парсер имени mp4» | прочее | задача на код |
| «как ИИ заменяет закупщиков» | тема | вопрос для аудитории, домен закупки + ИИ |
| «бутылочное горлышко в производстве» | тема | концепт ТОС |
| «https://youtube.com/watch?v=abc, опубликуй» | YouTube URL | URL правит |
| «Тема: 3 признака что финдир ворует» | тема | явный префикс |
| «нужен сценарий про тендеры в нефтянке, на 28 апреля 14:00» | тема + express | префикс «нужен сценарий» + дата |

## Что НЕ делает этот скилл

- Не генерирует сценарий сам (это работа `/video-script`)
- Не готовит посты сам (это работа `/publish-from-script`)
- Не делает технические задачи по репо
- Не валидирует тему («хорошая ли это идея?») — это работа целевого скилла на этапе уточнения
