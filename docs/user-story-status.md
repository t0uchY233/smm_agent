# User Story Status

Canonical tracker for the SMM Agent application audit.

Assumption: "functions" means the runnable user workflows and the top-level Python functions in `tools/*.py`. Test functions are excluded. Internal helpers are tracked as maintainer-facing user stories because their behavior affects the operator workflows.

Status legend:
- `PASS` - verified by current automated tests or direct compile/import evidence.
- `FAIL` - reproducible failure exists and is listed in the defect log.
- `TODO` - documented, but still needs direct story-level testing.
- `N/A` - support type or exception class with no standalone user behavior.

Baseline evidence on 2026-06-22:
- `python3 -m py_compile tools/render_video_visuals.py tools/read_docx.py tools/generate_cover.py tools/upload_images.py tools/validate_dzen_rss.py tools/wordpress_media.py` passed.
- Initial `python3 -m pytest -q` returned 74 passed, 5 failed.
- After fixes and expanded story tests, both `python3 -m pytest -q` and bare `pytest -q` return 112 passed.
