# Video Script — Генерация сценария для YouTube

## Overview

Шаг 0 в контент-пайплайне. Пользователь приходит с темой или идеей → в диалоге уточняем концепцию → генерируем сценарий по формату Tone of Voice → после утверждения выдаём телесуфлёр-текст.

**Pipeline:**
```
[Шаг 0] Тема + дата/время публикации → Сценарий → Телесуфлёр (DOCX с зашитой датой в имени)
[Шаг 1] Запись видео (пользователь)
[Шаг 2] Google Drive (имя YYYY-MM-DD-HHMM-alias.mp4) → n8n (YouTube scheduled + thumbnail + уведомление)
[Шаг 3] /publish-from-script <youtube_url> → читает DOCX → готовит контент → ставит в очередь Google Sheets
[Шаг 4] n8n Publisher Trigger (5 мин) → публикует в Telegram канал и блог в назначенное время
```

## Режимы запуска

**Режим диалога (по умолчанию):** пользователь даёт тему → обсуждаем угол подачи, ключевой кейс, цифры → генерируем сценарий.

**Express mode:** пользователь даёт тему + контекст в одном сообщении (например: "сценарий про бутылочное горлышко в закупках, кейс — металлургический завод, экономия 12 млн") → сразу генерируем сценарий без дополнительных вопросов.

## Workflow

### Step 1: Ввод и уточнение

1. Прочитать `tone-of-voice.md` из директории проекта (Read tool, НЕ спрашивать у пользователя)
2. Получить тему от пользователя
3. **Получить дату+время публикации** (обязательно).
   - В диалоговом режиме — спросить через AskUserQuestion: «Когда публикуем? Формат: YYYY-MM-DD HH:MM (MSK)».
   - В express mode — пользователь может указать в одном сообщении («сценарий про X, публикация 2026-05-01 14:00»).
   - **Валидация:** формат строго `YYYY-MM-DD HH:MM`, время не в прошлом (минимум now+15 мин). При ошибке — переспросить.
   - Сохранить как `pub_date` (YYYY-MM-DD) и `pub_time` (HHMM, без двоеточия — для имени файла).

**Если тема без деталей** (только заголовок/направление), задать 2-3 вопроса через AskUserQuestion:
- Какой конкретный кейс или ситуацию разберём? (пример из практики, с цифрами)
- Какая главная боль аудитории в этой теме?
- Есть ли конкретные цифры/суммы для примера?

**Если тема с деталями** — сразу к Step 2.

**Не задавать больше 3 вопросов.** Недостающие детали — додумать самостоятельно в стиле Веселкова (инженерный подход, реальные цифры из нефтегаза/закупок/производства).

### Step 2: Генерация сценария

Структура сценария (из tone-of-voice.md, формат YouTube/Rutube):

```
[ХУК — 0:00] (15-20 сек)
Провокационное начало: угроза потери денег, замена ИИ, обман в отрасли.
Конкретная цифра или факт, который бьёт по больному.

[ТЕЗИС — 0:20] (20-25 сек)
Что покажем в этом видео. Обещание конкретного результата с цифрой.
"Сегодня я покажу схему, как..."

[ТЕОРИЯ — 0:45] (≤2 мин)
Краткая подача концепции/принципа.
Метафоры из производства. Без академизма — только практика.
Максимум 2 минуты, иначе зритель уходит.

[КЕЙС — 2:45] (4-6 мин)
Разбор конкретного примера с цифрами.
Кто, что, сколько, за какой срок.
Пошаговый алгоритм: что сделал → что получил.

[РЕЗЮМЕ — 8:00] (30-60 сек)
Жёсткое резюме без долгих прощаний.
3 ключевых вывода пунктами.
CTA: "Ссылка в описании", "Смотри следующий выпуск".
```

**Хронометраж:** 8-12 минут итого. Указывать примерные тайм-коды.

**Стиль текста:**
- От первого лица ("я")
- Императивное "ты" к зрителю
- Метафоры из производства/инженерии
- Формула "Проблема → Решение → Деньги"
- Конкретные цифры: суммы в рублях, сроки, проценты
- Капслок для акцентов в ключевых словах
- Короткие предложения, рубленый ритм
- Без психологии и мотивации — только инженерия процессов

### Step 3: Утверждение

Показать сценарий пользователю через AskUserQuestion в компактном формате:

```
Сценарий: [НАЗВАНИЕ]
Хронометраж: ~N мин

[ХУК] первые 2 предложения...
[ТЕЗИС] первые 2 предложения...
[ТЕОРИЯ] краткое описание блока
[КЕЙС] краткое описание + ключевые цифры
[РЕЗЮМЕ] 3 пункта

Утверждаем? [OK / Изменить / Переделать]
```

**Если "Изменить"** — спросить что именно, поправить конкретный блок.
**Если "Переделать"** — спросить что не так, сгенерировать заново.
**Если "OK"** — перейти к Step 4.

### Step 4: Телесуфлёр

После утверждения — вывести чистый текст для телесуфлёра:

**Правила телесуфлёр-текста:**
- Убрать все тайм-коды и метки блоков ([ХУК], [ТЕЗИС] и т.д.)
- Убрать ремарки и комментарии
- Только чистый текст, который читается с суфлёра
- Разбить на абзацы по 2-3 предложения (удобно читать)
- Крупные смысловые блоки разделить пустой строкой
- Сохранить разговорный ритм: так, как говорят вслух, а не пишут

Вывести текст в чат — пользователь копирует его в приложение телесуфлёра.

### Step 5: Сохранить DOCX

**Всегда** после вывода телесуфлёр-текста — создать DOCX файл в `.tmp/teleprompter/`.

**Имя файла:** `YYYY-MM-DD-HHMM-alias.docx`, где:
- `YYYY-MM-DD-HHMM` — дата и время отложенной публикации (`pub_date` + `pub_time` из Step 1)
- `alias` — транслит заголовка через дефисы

Пример: `2026-05-01-1400-burovye-krs-7-prichin.docx`

**Важно:** имя файла = время публикации. По нему `/publish-from-script` потом найдёт DOCX и поставит посты в очередь Google Sheets на это же время. Имя видеофайла в Google Drive должно быть таким же (без `.docx`, с `.mp4`).

**Формат DOCX:**
- Шрифт: Arial 16pt (крупный, удобно читать с суфлёра)
- Межстрочный интервал: 1.4
- Заголовок: level 0, по центру
- **Под заголовком — строка `Публикация: YYYY-MM-DD HH:MM MSK`** (мелким серым шрифтом)
- Абзацы: 2-3 предложения, отступ после каждого 10pt
- Смысловые блоки разделены серым `— — —`
- Поля страницы: 2 см сверху/снизу, 2.5 см по бокам

Использовать `python-docx` через Bash. Не забывать `sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')` для Windows.

## Примеры тем

Для понимания контекста — типичные темы канала Веселкова:
- ТОС (теория ограничений) в управлении запасами
- Как ИИ заменяет закупщиков — и как на этом заработать
- Бутылочное горлышко в производстве — где теряешь деньги
- Инвестиции вне бутылочного горлышка — почему это слив бюджета
- Пенсионер с опытом 20+ лет: алгоритм выхода на 350К/мес

## Чего НЕ делать

- Не генерировать визуалы или thumbnail (это задача n8n)
- Не публиковать сценарий никуда (это рабочий документ)
- Не предлагать "улучшить" стиль — стиль определён в tone-of-voice.md
- Не делать сценарий длиннее 12 минут без явной просьбы
- Не добавлять мотивационные блоки, психологию, "мышление успеха"
